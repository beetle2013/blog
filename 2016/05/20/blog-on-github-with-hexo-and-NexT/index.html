blog-on-github-with-hexo-and-NexT

发表于 2016-05-20

前言

作为一名前端er,搭建个人博客有很多种选择。曾经我注册过CSDN之类的blog服务商，申请过自己的域名和使用wordpress组建blog,如今我选择Hexo+github的组合。不得不说，如今的方式可以让我更专注于写作，向大家推荐。

安装hexo

hexo使用是的Node.js，所以生成blog的速度很快，同时它支持我最喜欢的Markdown语法，hexo的安装非常简单，照着官网的说明就可以。 

1. $ npm install hexo-cli -g  全局安装hexo
2. $ hexo init blog  在当前目录生成blog目录，hexo已安装入其中
3. $ npm install  在blog目录下运行，加入依赖

之后日常是这样的

  • 新建 $ hexo new [layout] <title>
  • 写作 blog\source_posts 下的md文件
  • 生成 $ hexo generate blog\public目录为生成的blog站点
  • 本地运行 $ hexo server 想预览的话

其间出现一个问题，就是写好的文章加了标签，在首页点击标签 提示Cannot GET /blog/tags/,后来网上搜了一下，通过文章https://www.zhihu.com/question/29017171解决。

布署至github

github作为全世界最受欢迎的git仓库，吸引了全球的开源爱好者在此托管代码，同时它提供一个非常优秀的服务，那就是github.io的网页

  1. 注册github帐号，如果还没有的话
  2. 新建一个Repository,比如blog
  3. 在blog的Settings里找到 Launch automatic page generator 根据提示生成站点。
  4. git clone blog到本地 blog\public目录下
  5. git add public 下的所有文件

初始站点的内容可以不去理会，因为它会被替换。
非常关键的一点是github只会把你gh-pages分支下的文件发布到github.io上

之后写好文章就git commit 再push 爽不爽
